# Agent Decision Lab v0.2.0

Status: release candidate implemented locally; no npm publish, Git tag, or
GitHub Release has been performed.

## Theme

Version 0.2 is the transactional reliability and recovery release. It hardens
the existing local-first workflow before direct model or hosted integrations.

## Highlights

- Cross-worktree events are attributed to the invoking registered variant.
- Lab-scoped locking and atomic writes prevent parallel metadata corruption.
- `adl migrate --dry-run` previews v1-to-v2 migration before atomic apply.
- Git-affecting variant creation records an operation journal.
- `adl repair --dry-run` reports incomplete state without deleting or changing
  branches, worktrees, or metadata.
- Unknown CLI options fail before side effects.
- MCP tools publish explicit input schemas.
- The local UI uses a random launch token, same-origin POST checks, and a 1 MiB
  JSON body limit.

## Upgrade

```bash
adl doctor
adl migrate --dry-run
adl migrate
adl doctor
```

Keep a private copy of `.agent-lab/` before migration when the experiment is
important. Unknown future schemas fail closed.

## Verification

The release candidate must pass `npm run verify`, coverage, a three-worktree
parallel acceptance test, and `adl privacy audit --public-files --json` before
tagging. No npm publish is part of the local acceptance gate.
