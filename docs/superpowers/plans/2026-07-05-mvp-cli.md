# Agent Decision Lab MVP CLI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first usable `adl` CLI for local-first branching AI development experiments.

**Architecture:** The CLI is a dependency-free Node.js ESM package. Core logic is split into focused modules for argument parsing, Git state, file-backed storage, event logging, tree rendering, export rendering, and redaction. Tests use temporary Git repositories and Node's built-in test runner.

**Tech Stack:** Node.js 22+, ESM JavaScript, `node:test`, Git CLI, JSON, JSONL, Markdown.

## Global Constraints

- Keep behavior local-first and transparent.
- Do not upload data automatically.
- Do not push, force-push, delete branches, delete worktrees, or rewrite history.
- Keep real private experiment data out of this repository.
- Treat prompt text, model output, command output, paths, and metadata as untrusted input.
- Prefer structured JSON and JSONL over ad hoc string storage.
- Write failing tests before production code for behavior changes.
