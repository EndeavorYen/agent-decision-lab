# Agent Decision Lab v0.2 Reliability and Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver v0.2 with correct cross-worktree attribution, a transactional and recoverable file store, shared CLI/UI/MCP contracts, and a freshly verified release surface.

**Architecture:** Keep the inspectable `.agent-lab/` layout, but route mutations through a cross-process lab lock and atomic file writes. Add schema validation, explicit v1-to-v2 migration, and an operation journal around Git side effects. Reuse declarative command contracts across CLI and MCP, and protect the local UI with a launch token and same-origin checks.

**Tech Stack:** Node.js ESM, built-in `node:test`, Git CLI, dependency-free filesystem and HTTP modules.

## Global Constraints

- Node.js remains `>=22` and the runtime dependency count remains zero.
- `.agent-lab/` remains local and private by default.
- No automatic branch/worktree deletion, push, tag, publish, or hosted upload.
- Every production behavior change follows RED, GREEN, REFACTOR.
- Public examples and tests use synthetic data only.

```js
export function resolveInvocationVariant(store) {
  if (store.invocationPath === store.labRoot) return null;
  return store.variants.find((variant) => (
    variant.worktreePath && resolve(variant.worktreePath) === resolve(store.invocationPath)
  )) ?? null;
}
```

store.config.activeVariantId ?? null` when appending events.
