# v0.1.0 Release, Adapters, and Onboarding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Agent Decision Lab v0.1.0 with provider-neutral adapter/plugin recipes, production onboarding, a Reality Slap Skill live case study, and a GitHub Release.

**Architecture:** Keep the CLI dependency-free and local-first. Adapters are documented integration recipes that scaffold redacted, target-local operator guides; they do not execute model providers or inject hidden prompts. Onboarding is enforced by a `doctor` command, docs, tests, and release notes.

**Tech Stack:** Node.js ESM, built-in `node:test`, Git CLI, GitHub CLI for the final release.

## Global Constraints

- Tool is open. Experiments are private.
- Do not commit raw `.agent-lab/` experiment state.
- Do not add direct model orchestration or provider dependencies in v0.1.0.
- Keep release publication to GitHub tag/release only; do not publish to npm.
- Run `npm run verify` before tagging.
