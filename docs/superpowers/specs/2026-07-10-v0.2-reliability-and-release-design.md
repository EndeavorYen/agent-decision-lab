# Agent Decision Lab v0.2 Reliability and Release Design

## Objective

Make Agent Decision Lab safe for parallel variant work before adding more
provider, TUI, or hosted features. Version 0.2 must preserve the local-first,
dependency-free, inspectable file store while preventing event
misattribution, concurrent metadata corruption, and silent partial Git
operations.

## Non-Negotiable Boundaries

- The tool remains provider-neutral and does not execute model-generated
  commands automatically.
- Raw experiment data remains under `.agent-lab/` and private by default.
- No branch, worktree, or experiment data is deleted automatically.
- The implementation remains dependency-free on Node.js 22 or newer.
- Existing v1 experiment stores remain readable and can be migrated to v2.
- Unknown newer schemas fail closed with a clear operator message.

## Store Architecture

All metadata mutations pass through one lab-scoped transaction boundary. The
boundary uses an atomic lock directory so separate ADL processes and variant
worktrees serialize writes. JSON documents are written to a temporary file in
the destination directory and atomically renamed into place. Event JSONL
appends happen while the same lock is held.

The v2 store keeps the existing human-readable directory shape. It adds a
monotonic `revision` to mutable aggregate documents and an `operations/`
journal for mutations that combine Git side effects with metadata writes.
Every store read validates the schema and cross-record references before the
data is exposed to commands, the UI, or MCP.

## Variant Attribution

`loadCurrentStore()` records both the owning lab root and the checkout from
which the command was invoked. Event attribution follows this order:

1. Explicit `--variant` or MCP/UI variant argument.
2. The registered variant whose `worktreePath` matches the invocation
   checkout.
3. The lab-wide active variant when invoked from the base checkout.
4. Fail closed if no unambiguous variant exists for a variant-scoped action.

This prevents an event created in one worktree from inheriting the last
variant selected by another process.

## Operation Recovery

Git-affecting workflows write a prepared operation record before creating a
branch or worktree. The record contains the operation id, intended branch,
worktree, base commit, and metadata targets. Successful completion marks the
record `complete`; exceptions mark it `failed`; process termination leaves it
`prepared`.

`adl doctor` reports prepared or failed operations. `adl repair --dry-run`
compares the journal with Git and metadata state and emits deterministic,
non-destructive repair actions. Version 0.2 does not automatically delete or
rewrite Git state.

## Command Contracts

CLI, local UI, and MCP share explicit input contracts. CLI option validation
rejects unknown flags and suggests the closest supported flag. MCP publishes
real JSON Schemas for every tool. The local UI requires a random launch token
for API and event-stream requests, validates same-origin requests when an
Origin header is present, and limits JSON request bodies.

## Release Contract

The v0.2 release gate includes the existing test, smoke, privacy, diff, and
pack checks plus parallel-writer, worktree-attribution, schema migration,
operation recovery, CLI typo, MCP schema, and UI authorization tests. Release
metadata, roadmap, handoff, security guidance, changelog, and contributor
guidance must describe the same v0.2 behavior.

## Acceptance Criteria

- Twenty concurrent writers complete without corrupting JSON or losing
  records.
- Events created without `--variant` in registered worktrees are attributed to
  those worktrees' variants.
- Every injected failure around a Git-affecting mutation produces a detectable
  operation record and a non-destructive repair plan.
- A synthetic v1 store migrates to v2; malformed or newer unsupported schemas
  fail closed.
- Unknown CLI options fail, MCP tool schemas are non-empty, and unauthorized UI
  requests are rejected.
- `npm run verify`, coverage, privacy audit, and a three-variant parallel smoke
  all pass from a clean branch.
